The wasmCloud host communicates with WebAssembly guests using the [WebAssembly component model](https://github.com/WebAssembly/component-model). You can read more about components in our [Platform Overview](https://wasmcloud.com/docs/v1/concepts/components/).

### Reference

Previously wasmCloud had a purpose-built ABI called [wasmbus](https://wasmcloud.com/docs/0.82/hosts/abis/wasmbus/) for communicating with Wasm guests, but it did not provide portability between runtimes or a standard way to communicate between host and guest.
