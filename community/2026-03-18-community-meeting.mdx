### A quick update on the JS Ecosystem

- A new StarlingMonkey release
  - Great job by Tomasz on updating to the latest version of StarlingMonkey
  - https://github.com/bytecodealliance/StarlingMonkey/releases
- componentize-js is being rewritten
  - Joel has ported componentize-js to Rust and implemented guest p3 (async, streams, etc)
    - https://github.com/dicej/componentize-js
  - This uses `wit-dylib`, a crate that can be used to implement components for interpreted languages (think Python, Ruby, etc)
    - https://github.com/bytecodealliance/wasm-tools/tree/main/crates/wit-dylib
- componentize-qjs
  - Tomasz has been working on getting quickjs-ng working with wit-dylib
  - https://github.com/andreiltd/componentize-qjs
  - At some point, componentize-js *should* become a multi-guest-runtime project
- Jco p3 host suport
  - Jco p3 host support now has streams merged, with work right now testing out lowering/lifting for all the types
  - Only futures left
  - Check out the tests for streams: https://github.com/bytecodealliance/jco/blob/main/packages/jco/test/p3/stream.js
  - New version of Jco is out so you can try things today (and file bugs if something is broken!)
    - https://www.npmjs.com/package/@bytecodealliance/jco/v/1.17.2
- Someone vibecoded a fully JS host implementation
  - Jelles has done amazing work including a sweet demo of Go running in the browser
  - https://github.com/jellevandenhooff/wcjs
  - We want to get this merged upstream in the Jco project someday, so we can be a multi-host-runtime project
  - AIs can now output WebAssembly host implementations, including custom parsers and visitors for operations (!!)
